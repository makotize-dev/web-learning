暗号資産ページで学ぶ HTML → CSS → JS
原理

HTMLはツリー構造で
「意味」を記述する

HTMLはページの「骨格」です。スタイルも動作も一切持たず、コンテンツが何であるか（意味） だけを表現します。ブラウザはこれをDOMツリーとして解釈します。

1
タグは暗記しない。 「これは見出しか？本文か？リストか？」という意味から演繹してタグを選ぶ。
2
入れ子＝親子関係。 <section>の中に<h2>があるのは「セクションが見出しを含む」という意味の表明。
3
ブラウザのデフォルトスタイルがある。 右のプレビューはCSSゼロの状態。見た目は素朴でも構造は完成している。
<!-- HTMLのみ。CSSもJSも一切なし -->

<header>
  <h1>暗号資産とは</h1>
  <p>Web3時代の新しいお金の形</p>
</header>

<main>
  <section>
    <h2>定義</h2>
    <p>暗号資産とは、ブロックチェーン技術を
    基盤とするデジタル資産です。</p>
    <div>
      <span>分散型</span>
      <span>ブロックチェーン</span>
      <span>P2P</span>
    </div>
  </section>

  <section>
    <h2>主な種類</h2>
    <table>
      <tr><th>名称</th><th>特徴</th></tr>
      <tr><td>Bitcoin</td>
          <td>最初の暗号資産</td></tr>
      <tr><td>Ethereum</td>
          <td>スマートコントラクト</td></tr>
    </table>
  </section>
</main>

<!-- ↑ これだけで「構造」は完成 -->
原理

CSSはカスケードと
ボックスモデルで制御する

CSSはHTMLの構造に視覚的な意味を与えます。「どのルールが勝つか」は詳細度（Specificity）で決まり、「どこに配置するか」はボックスモデルで決まります。

1
カスケードの法則： 同じプロパティが競合したとき、!important > インライン > ID > クラス > タグ の順で勝者が決まる。
2
ボックスモデル： すべての要素は content + padding + border + margin の箱。レイアウトの問題はこの箱のどこかに原因がある。
3
Flexboxは配置の抽象化： 子要素を横・縦に並べる制御をFlexboxに委譲できる。親にdisplay:flexを書くだけで子が整列する。
/* 詳細度: タグ(0,0,1) < クラス(0,1,0) */

.page-header {
  background: linear-gradient(
    135deg, #1e3a5f, #0d1f3c
  );
  color: white;
  padding: 32px 24px; /* ボックスモデル */
}

/* Flexboxで横並び */
.badges {
  display: flex;
  gap: 8px;
  flex-wrap: wrap;
}

.badge {
  padding: 4px 12px;
  border-radius: 20px;
  background: #ebf5fb;
  border: 1px solid #bee3f8;
}

/* テーブルの縞模様
   :nth-child は疑似クラス → 詳細度(0,1,0) */
tr:nth-child(even) td {
  background: #f7fafc;
}
原理

JavaScriptはイベントに
反応してDOMを変更する

JSのすべての操作は ①イベントを待ち受ける → ②DOMを選択する → ③状態を変更する の3ステップに還元できます。これが分かれば「なぜ動くか」が演繹できます。

1
DOMはツリー。 document.querySelector() はCSSセレクタと同じ記法で要素を「選ぶ」。選べなければ何もできない。
2
イベントリスナーは「待ち受け係」。 addEventListener('click', fn) は「クリックが来たらfnを呼べ」という命令。イベントが来るまでfnは実行されない。
3
classListでCSSに委譲。 JSは直接スタイルを書かず、クラスを付け外しするだけでよい。見た目の制御はCSSに戻す設計が保守性が高い。
// ① イベント待ち受け（タブ切り替え）
const buttons = document.querySelectorAll('.tab-btn');

buttons.forEach(btn => {
  btn.addEventListener('click', () => {

    // ② DOM選択 → クラス除去
    document.querySelectorAll('.tab-btn')
      .forEach(b => b.classList.remove('active'));
    document.querySelectorAll('.tab-panel')
      .forEach(p => p.classList.remove('active'));

    // ③ 状態変更（classListでCSSに委譲）
    btn.classList.add('active');
    const target = btn.dataset.tab;
    document.getElementById(target)
      .classList.add('active');
  });
});

// バッジクリックでツールチップ表示
document.querySelectorAll('.badge').forEach(b => {
  b.addEventListener('click', () => {
    const tip = b.nextElementSibling;
    tip.classList.toggle('show');
  });
});
原理

Promiseは「未来の値の約束」
async/awaitはそれを同期的に書く

JavaScriptはシングルスレッド。ネットワーク通信の完了を待ちながら止まることはできません。代わりに Promise（後で値が届く約束）を返し、届いたときに処理を続けます。async/await はその書き方を直感的にした構文糖です。

1
fetchはPromiseを返す。 fetch(url) は即座に Promise を返す。データはまだない。await で解決を待つまで次の行は実行されない。
2
async関数の中でのみawaitが使える。 async を付けた関数の戻り値は自動でPromiseにラップされる。呼び出し側もasyncにする必要がある。
3
try/catchでrejectを捕捉。 ネットワークエラーなどPromiseが失敗すると reject される。try/catch がないとエラーが握りつぶされ、画面が無反応になる。
// ① 非同期関数の定義（asyncキーワード）
async function fetchPrices() {

  // fetch()はPromiseを返す → awaitで解決を待つ
  const res = await fetch(
    'https://api.example.com/v1/prices'
  );

  // JSON変換もPromise（awaitが必要）
  const data = await res.json();

  return data; // Promiseでラップされて返る
}

// ② 呼び出し側：エラーハンドリング
async function loadData() {
  try {
    const prices = await fetchPrices();
    renderUI(prices); // データが来てからDOM更新

  } catch (err) {
    // reject → ここへ（ネットワークエラー等）
    showError(err.message);
  }
}

// ③ イベントから呼び出す
btn.addEventListener('click', loadData);
原理

@keyframesは「状態遷移の
コマ撮り」を定義する

CSSアニメーションは ①コマを定義（@keyframes）→ ②要素に適用（animation） の2ステップ。transition が「状態変化への反応」であるのに対し、animation は「クラスを付けるだけで自律的に動く」のが違いです。

1
@keyframesは定義だけ。 @keyframes 名前 { 0% ... 100% } はアニメーションの「コマ」の定義に過ぎない。要素に animation プロパティで適用して初めて動く。
2
transitionとanimationは別物。 transition はCSSプロパティの変化に補間を加える。animation は@keyframesに沿って繰り返し・途中停止など複雑な動きが作れる。
3
transformはレイアウトを壊さない。 translate / scale / rotate は占有スペースを変えず視覚的に動かす。top/left で動かすとリフローが走るが、transformは軽量でGPU処理になる。
/* ① @keyframes でコマを定義 */
@keyframes price-up {
  0%   { background: #ffffff; }
  65%  { background: #16a34a;
          color: white;
          transform: scale(1.03); }
  100% { background: #ffffff;
          transform: scale(1); }
}

/* ② animation プロパティで適用 */
.card.up {
  animation: price-up 0.9s ease;
  /*          名前     時間  イージング */
}

/* ③ transform: レイアウトを壊さず変形 */
.badge {
  transition: transform 0.15s ease;
}
.badge:hover {
  /* 複数のtransformは半角スペースで連結 */
  transform: translateY(-4px) scale(1.08);
}

分散型

P2P
銀行などの仲介者なしに、個人間で直接取引（Peer to Peer）が可能です。
スマートコントラクト
トリガーに対応してプログラムが実行されます。
